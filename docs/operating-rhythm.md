# Operating Rhythm (Part-time, Stability-first)

## Weekly Cadence
- One fixed **stability window** per week: bug fixes and performance only.
- Feature changes are paused during the stability window.

## Time Allocation
- 70%: reliability + performance work
- 20%: small feature improvements
- 10%: UI polish

## Feedback Intake
- Keep a single feedback backlog (one document or one issue list).
- Triage priority:
  1. data safety/security
  2. performance/stability
  3. feature requests
